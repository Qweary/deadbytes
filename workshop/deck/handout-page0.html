Page 0 — The Codes That Open the Door, in Plaintext

This is page 0 of the DataFlash chip from a real Alarm Lock T2/T3 — the same read you’ll take at the bench. Every code that opens this door sits here in plaintext on a $0.50 flash chip; nothing is encrypted, nothing is hidden. The fields are stored column-major — all the slots’ first code bytes, then all their second, then their third — so a single user’s code reads down the columns, not across a row. That column-major layout is the diagonal you’ll trace through the grid below.

HOW TO READ THE DIAGONAL
A single user’s five bytes are not next to each other — they sit one band apart, at offsets 0x01+N, 0x33+N, 0x65+N, 0x97+N, 0xC9+N (each band 50 bytes / 0x32). In a 16-byte-wide hex view, stepping +0x32 to the next field lands you three rows down and two columns to the right (0x32 = 50 = 3×16 + 2). Each band starts one slot later than the one before, so the structure marches down and to the right as you scan — that’s the “diagonal.” To recover a code, take the bytes at 0x01+N, 0x33+N, and 0x65+N and decode the three as packed BCD.
The 0 → 0xB rule: a decimal digit 0 is stored as nibble 0xB, so a zero reads as “B” in the hex (slot 0 = 010101 stores as B1 B1 B1) — it is the nibble value 0xB, not the letter B.
